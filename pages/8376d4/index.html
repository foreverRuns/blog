# QT shadow build

Shadow building means building a project in a separate directory, the build directory. The build directory is different from the source directory. One of the benefits of shadow building is that it keeps your source directory clean, which makes it faster to switch between build configurations. Therefore, shadow building is the best practice if you need many build configurations for a single set of source files. Shadow builds are also widely known as out-of-source builds to separate them from in-source builds or in-tree builds.
